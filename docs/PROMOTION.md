## 完整平台简介

方界造模是一套面向 Minecraft Java 版 Mod 作者、服务器开发者、地图制作者和独立创作者的 Blockbench 全流程智能资产生产系统。

用户不需要先熟练掌握 Blockbench，也不必一开始就理解骨骼、UV、动画控制器、粒子或声音事件。只需要用中文描述目标，方界造模就会把模糊创意逐步整理成可确认、可生产、可验证的模型方案。

它不会在没有询问需求、没有概念图确认的情况下直接生成模型。正式建模前，系统会确认模型类别、游戏用途、相关资产、纹理精度、动画、损毁、粒子、音效和 Mod 接入目标，并生成三套受 Blockbench 实际能力约束的概念方案。用户明确选择和批准后，系统才进入生产。

最终交付不只是一个 `.bbmodel`，而是一套身份清晰、目录独立的资产包：模型源文件、纹理、骨骼动画、音频、粒子、运行时导出物、验证报告和版本记录。

## 核心卖点

### 1. 用户批准优先

路径、资产范围、需求、概念、建模、音效和 Mod 接入分别确认。AI 不能把沉默、偏好或“你看着办”自动解释为生产批准。

### 2. 概念图受实际建模能力约束

概念图不是脱离生产的电影海报。方块几何、纹理密度、骨骼原点、旋转范围和运行时光效边界在生图前就被锁定。

### 3. 完整动画系统

从待机、启动、蓄力、攻击到冷却、归位、受击、死亡和损毁，动画按照状态、转换、中断和事件进行设计，而不是只放几个孤立关键帧。

### 4. 模型、粒子、声音和 Mod 联动

动画事件可以触发粒子、音效、投射物与游戏逻辑。中文或数字音频文件会先映射为英文事件名，并绑定到明确模型和动作。

### 5. 一个模型一个工作区

主体、投射物、掉落物和独立残骸分别拥有稳定身份和独立目录，避免纹理、动画、音频和导出物交叉污染。

### 6. 证据驱动交付

重新打开、结构校验、动画检查、路径隔离、构建和游戏内验证分别留下证据，不把“工具执行成功”包装成“成品已经可用”。

## 真正改变行业的行为

方界造模的突破不应被描述成“一键取代建模师”。它真正试图改变的是协作方式：

> **把原本依赖个人经验的需求沟通、概念评审、返工和交付，变成用户能够参与确认、机器能够执行、结果能够验证的生产流程。**

用户不再需要先学会整套软件，才有资格准确表达创意；AI 也不能跳过用户，擅自决定最终设计。
