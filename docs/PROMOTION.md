## 一句话介绍

方界造模是一套给 Minecraft Java 创作者使用的 Codex Skill。它会先问清需求、生成三套 Blockbench 可实现的方案图，等用户选定后，再完成模型、贴图、动画、粒子、音效和 Mod 接入。

## 大白话版本

你不会 Blockbench 没关系。告诉它你想做什么，它会像一个建模项目经理一样，把外观、精度、动画、音效、损毁效果和保存位置问清楚。然后先给你三套图，你选好以后它才开始正式建模。

最关键的是：它不会在你没同意的时候擅自开工，也不会把水晶塔的音效绑到三头狼身上。✅

## 它解决了哪些老问题？

### 问题一：AI 没问清楚就直接做

现在：需求没确认完，只能提问和出方案，不能正式建模。

### 问题二：概念图很漂亮，Blockbench 根本还原不了

现在：方案图必须遵守方块结构、真实转轴、实际纹理和 Blockbench 能力，尽量让“图里看到的”和“软件里做出的”接近。

### 问题三：动画只会生硬地动

现在：动画要写启动、攻击、命中、等待、冷却和归位，还要检查所有中间帧有没有穿模。

### 问题四：模型、音效和粒子互相串了

现在：一个模型一个文件夹、一个身份编号；绑定前必须显示模型名和事件表。

### 问题五：生成成功就说完成

现在：文件要能打开，动画要能播放，Mod 要能运行，并留下截图和报告。

## 三个宣传例子

- 🐺 **机械三头狼 Boss**：三个头分别攻击，红眼发光，死亡后留下破损残骸。
- ⚡ **水晶能量塔**：能量从下往上输送，护盾围绕水晶公转，停火 5 秒后归位。
- 🗡️ **魔法巨剑**：符文按顺序点亮，蓄力完成出现粒子，耐久耗尽切换断裂模型。
