# 宣传图提示词：复制就能用 🎨

这套提示词一共七张图。先做 P00，确认整体颜色和模型风格；后面的 P01 到 P06 都把 P00 当参考图使用，这样不会每张图都长得不一样。

## 生成前先准备

- 画面比例：16:9；
- 推荐尺寸：至少 1920×1080；
- 统一风格：黑铁机械结构、青蓝能量光、暗黑游戏界面；
- 统一模型：以水晶能量塔、机械三头狼和魔法巨剑作为实际例子；
- 重要要求：模型必须像 Blockbench 里真的能做出来，不能用电影级特效把结构遮住。

## P00：视觉基准

**用途：** 先把整套图片的颜色、材质和方块精度定下来。

**直接复制：**

~~~text
制作一张 Minecraft Blockbench 模型项目的视觉基准图。暗黑游戏风，深灰黑背景，黑铁机械材质，青蓝色能量光作为少量重点。画面同时展示三个可实现的方块模型：机械三头狼 Boss、水晶能量塔、三阶段魔法巨剑。所有结构由清楚的立方体、长方体和有限斜面组成，保留明显像素纹理、装甲接缝、铆钉、磨损和方块阴影。使用接近 Blockbench 透视视图的 3/4 观察角度，能看清模型轮廓、关节、转轴和部件间距。不要真人，不要写实圆滑曲面，不要烟雾遮挡，不要夸张景深，不要把画面做成电影概念艺术。画面必须让人直观看出最终 Blockbench 模型大概是什么水平。
~~~

## P01：GitHub 首页主视觉

**用途：** 第一眼告诉别人，这个 Skill 能从想法做到游戏模型。

~~~text
以 P00 为视觉参考，制作横版 GitHub 项目主视觉。中央展示一座完成度高的 Blockbench 水晶能量塔，左侧用较小画面展示三套概念方案 A、B、C，右侧展示 Blockbench 方块模型和 Minecraft 游戏内效果。标题区域预留清楚空间，主标题为“方界造模”，副标题为“先看懂，后建模；先确认，再生产”。黑铁、青蓝能量、暗黑游戏风，结构清楚，纹理细节真实但不写实。避免电影级烟雾、连续曲面、虚假的高级渲染和无法还原的复杂结构。
~~~

## P02：先确认，再生产

**用途：** 把最重要的工作方式画出来。

~~~text
以 P00 为参考，制作一张三段式流程图。第一段是中文对话框，用户提出“机械三头狼 Boss”；第二段并列展示 A、B、C 三个明显不同但都能在 Blockbench 实现的方块方案；第三段是用户选中 B 后才进入 Blockbench 正式建模。用清楚的箭头表现“说想法 → 三套图 → 用户批准 → 正式制作”。页面像专业游戏工具说明，不要卡通人物，不要假数据，不要过度装饰。所有模型保留像素纹理和可实现结构。
~~~

## P03：概念图和实际模型对照

**用途：** 证明方案图不是随便画的。

**注意：** 这一张最好上传真实 Blockbench 截图作为第二张参考图。

~~~text
以 P00 和真实 Blockbench 截图为参考，制作左右对照图。左边是批准后的水晶能量塔方案图，右边是 Blockbench 软件里的最终模型视图。两边必须保持相同轮廓、比例、黑铁材质、青蓝水晶、护盾板数量和位置。中间用细线标出转轴、骨骼、纹理尺寸和旋转半径。重点展示“看到的方案”和“实际做出的模型”很接近。不要美化成软件无法达到的渲染，不要隐藏结构。
~~~

## P04：完整动画

**用途：** 展示动画不是简单抖一下。

~~~text
以 P00 为参考，制作水晶能量塔动画分镜。按顺序展示六个清楚阶段：待机、启动、能量从下往上输送、护盾围绕水晶中心公转、发射能量弹、停火后等待 5 秒再减速归位。额外加入损毁状态，显示水晶破裂和塔身熄灭。用时间轴和关键帧标记表现动作顺序。两块护盾保持相同朝向、固定间距，不接触水晶，不重叠，不穿模。模型必须保持 Blockbench 方块结构。
~~~

## P05：模型、粒子、音效和 Mod 联动

**用途：** 解释真正进入游戏需要哪些部分。

~~~text
以 P00 为参考，制作一张清楚的系统联动图。中心是水晶能量塔模型，四周分别展示动画事件、青蓝粒子、编号音效文件、能量弹和 Minecraft Java Mod 事件。用有方向的连线表示正确顺序：启动 → 能量上升 → 水晶发光 → 发射 → 命中 → 冷却。所有标签使用简短中文，不展示程序员看不懂的假代码，不堆无意义图标。整体像游戏开发工具的专业说明页。
~~~

## P06：一个模型一个文件夹

**用途：** 展示多模型不会互相搞混。

~~~text
以 P00 为参考，制作一张项目交付图。画面并列展示三个独立工作区：mecha_cerberus、crystal_tower、arcane_greatsword。每个工作区分别包含 model、textures、animations、audio、particles、previews、mod、evidence 文件夹。用颜色和连线强调每个模型的动画、声音和粒子只属于自己的身份编号。右侧展示最终交付包、预览图和验证报告。暗黑游戏工具风，信息清楚，不能像普通文件管理器截图，也不要混放文件。
~~~

## 每张图都加的反向要求

~~~text
不要写实圆滑曲面，不要电影级烟雾，不要强景深，不要过曝，不要塑料玩具感，不要低清晰度，不要结构混乱，不要错误透视，不要随机增加零件，不要让护盾和水晶穿模，不要把左右部件方向做反，不要伪造 Blockbench 界面，不要乱码文字。
~~~

## 推荐生成顺序 ✅

1. 生成 P00，先确认黑铁、青蓝能量和方块精度；
2. 用 P00 作为参考生成 P01、P02；
3. 准备真实 Blockbench 截图后生成 P03；
4. 生成 P04、P05、P06；
5. 统一检查模型轮廓、颜色、文字和部件数量；
6. 最后压缩图片并上传到 assets/promo。

先把 P00 做对，后面会省下大量返工。Enjoy creating! ✨
